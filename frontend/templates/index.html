System Features
Cheating Detection

Two-strike system with automatic warnings and termination. First offense: 50% mark deduction. Second offense: immediate termination.

Time Synchronization

Berkeley algorithm ensures all student clocks are synchronized with the server, accounting for network delays and clock skew.

Mutual Exclusion

Ricart-Agrawala algorithm ensures only one process can access the marksheet critical section at a time.

Load Balancing

Round-robin distribution of read requests across multiple server replicas for improved performance and reliability.

Data Replication

Master-slave replication with automatic failover and consistency guarantees using version counters.

Conflict Resolution

Lamport timestamps and lexicographic ordering resolve conflicts between simultaneous manual and auto submissions.

Quick Start
  1. Start the System: Run the coordinator server and load balancer
  2. Register Students: Use the Teacher Dashboard to register student roll numbers
  3. Start Exam: Begin the exam session for all registered students
  4. Monitor: Watch real-time updates and manage the exam session
  5. End Exam: Terminate the exam and generate reports
Tip: Check the demo simulation script for automated testing of all features.
